# 第9章：基于代理的模型 (Agent-Based Models)

> 本章从"基于规则的模型"转向**基于代理的模型**——代理收集信息、做出决策、采取行动。重点探讨个体决策与系统行为之间的**不可预测关系**，以及**涌现**这一复杂性科学的核心概念。

---

## 9.1 Schelling 隔离模型 (Schelling's Model)

### 背景

1969 年 Thomas Schelling 发表《Models of Segregation》，提出种族隔离的简单模型。

### 模型规则

- **网格**：每个细胞代表一栋房子，约 **10% 为空置**
- **两种代理**：红色和蓝色，数量大致相等
- **邻域**：周边 8 个相邻细胞（Moore 邻域）
- **幸福条件**：邻居中**同类比例 ≥ 阈值**（如至少 2 个同类邻居）
- **运行方式**：随机选代理 → 若不满 → 随机迁至空置房

### 惊人结果

- 从随机初始状态开始，**同类聚类几乎立即形成**
- 聚类随时间增长和合并，最终形成少数大规模同类聚居区
- 关键洞察：代理本身**毫无种族歧视**——他们只是不愿成为极少数派
- 即便只是**轻度排外**（需要 ≥30% 同类），最终隔离程度高达 **76%-82%**

> **不可以从结果反推原因**：观察到现实城市的隔离现象，不能直接推断个体存在种族歧视。

## 9.2 Schelling 模型的实现

### 初始化

```python
class Schelling(Cell2D):
    def __init__(self, n, p):
        self.p = p                          # 同类邻居比例的阈值
        choices = [0, 1, 2]                 # 0=空, 1=红, 2=蓝
        probs = [0.1, 0.45, 0.45]          # 10%空, 45%红, 45%蓝
        self.array = np.random.choice(choices, (n, n), p=probs)
```

### 核心步骤

```python
# 1. 创建布尔数组
red = a==1; blue = a==2; empty = a==0

# 2. 用 correlate2d 统计每个位置的邻居
options = dict(mode='same', boundary='wrap')
kernel = np.array([[1, 1, 1],[1, 0, 1],[1, 1, 1]], dtype=np.int8)
num_red = correlate2d(red, kernel, **options)
num_blue = correlate2d(blue, kernel, **options)
num_neighbors = num_red + num_blue

# 3. 计算同类邻居比例
frac_red = num_red / num_neighbors
frac_blue = num_blue / num_neighbors
frac_same = np.where(red, frac_red, frac_blue)
frac_same[empty] = np.nan

# 4. 识别不满代理并移动
unhappy = frac_same < self.p
unhappy_locs = locs_where(unhappy)    # 不满代理的坐标列表
# 随机选空房移动
for source in unhappy_locs:
    dest = empty_locs[random_index]
    a[dest] = a[source]
    a[source] = 0
```

**关键 NumPy 技巧**：

| 技巧 | 说明 |
|------|------|
| `np.where(red, frac_red, frac_blue)` | 按元素选择：red 为 True 取 frac_red，否则取 frac_blue |
| `np.nonzero(condition)` | 返回所有非零元素的坐标元组 |
| `boundary='wrap'` | 边界环绕：上边接底边，左边接右边 |
| `np.nanmean(frac_same)` | 忽略 NaN 计算平均值 → 隔离程度 |

## 9.3 隔离动态 (Segregation Dynamics)

n=100，不同 p 值的隔离程度（同类邻居平均比例）：

| 时间步 | p=0.3 | 说明 |
|--------|-------|------|
| 初始 | **50%** | 随机分布 |
| 2 步 | **65%** | 聚类快速形成 |
| 10 步 | **76%** | 高度隔离 |
| 稳态 (p=0.4) | **82%** | 大多数代理完全没有异色邻居 |

**核心教训**：个体仅要求 30% 同类邻居即可满意 → 系统层面产生 **75%+ 的隔离**。个体意图与系统结果之间存在**巨大的、不可预见的差距**。

## 9.4 Sugarscape

### 模型概述

Epstein & Axtell (1996) 提出的"人工社会"模型，用于经济学和社会科学实验。

**环境**：
- 2-D 网格，每个细胞有**容量 (capacity)**：两个高糖区域（容量 4），同心环递减（3→2→1）
- 糖每步**再生 1 单位**，不超过容量上限

**代理属性**（初始 400 个，随机值）：

| 属性 | 范围 | 含义 |
|------|------|------|
| **Sugar（糖）** | 5~25 均匀分布 | 初始财富 |
| **Metabolism（代谢）** | 1~4 均匀分布 | 每步消耗的糖量 |
| **Vision（视力）** | 1~6 均匀分布 | 能"看到"多远距离的糖 |

### 代理行为（每步）

1. **观察**：沿 4 个方向各看 k 格（k = vision），选择**糖最多的空格**
2. **移动**：到达目标格，**收获**该格的糖
3. **消耗**：减去代谢所需糖量 → 若为负则**饿死移除**
4. 环境**再生**：每格 +1 糖（不超过容量）

### 结果

- 代理向高糖区域移动，高 vision 者最快到达
- 低 vision 者滞留在平原上随机徘徊
- 初始 400 人 → 稳态约 **250 人**（承载能力）
- 生存者特征：**高视力 + 低代谢的"幸运者"**，一旦站稳脚跟可无限积累糖

## 9.5 财富不平等 (Wealth Inequality)

### 引入有限寿命

- 年龄每步 +1，寿命从 **60~100 均匀分布**中随机选取
- 死亡（饿死或老死）后由**新随机代理替代** → 总人数恒定
- 250 人运行 500 步

### 财富分布特征

| 百分位 | 糖积累量 |
|--------|---------|
| 25th | ≈ 10 |
| 50th（中位数）| ≈ 20 |
| 75th | ≈ 40 |
| 最高值 | > 150 |

- 分布**右偏**，近似**对数正态分布**（重尾）
- 与现实世界财富分布一致：**几乎所有国家和全球的财富分布都是重尾的**
- 糖的再生率和简单经济规则 → **不平等是许多经济体的固有特征**，即便在极简模型中也难以避免或消除

## 9.6 Sugarscape 实现概要

### Agent 类

```python
class Agent:
    def step(self, env):
        self.loc = env.look_and_move(self.loc, self.vision)
        self.sugar += env.harvest(self.loc) - self.metabolism
        self.age += 1
```

### Sugarscape 类（继承 Cell2D）

```python
class Sugarscape(Cell2D):
    def step(self):
        random_order = np.random.permutation(self.agents)   # 随机顺序
        for agent in random_order:
            self.occupied.remove(agent.loc)                   # 标记离开
            agent.step(self)                                  # 代理行动
            if agent.is_starving():                           # 饿死移除
                self.agents.remove(agent)
            else:
                self.occupied.add(agent.loc)                  # 标记占据
        self.grow()                                           # 糖再生
        return len(self.agents)
```

**关键 NumPy 函数**：

| 函数 | 用途 |
|------|------|
| `np.random.permutation` | 随机排列代理顺序 |
| `np.vstack` | 构建可视位置数组 |
| `np.hypot` | 计算距离 |
| `np.digitize` | 初始化容量 |
| `np.argmax` | 选择糖最多的位置 |

## 9.7 迁徙与波行为 (Migration and Wave Behavior)

### 波现象

- 所有代理初始聚集在**左下角** → 快速涌向最近的高糖山峰
- 代理数超过单峰承载能力 → 糖被耗尽 → 代理被迫迁移
- **高视力者**穿越山谷向东北方向传播 → 形成**波前 (wave front)**
- 身后留下**空细胞带** → 后续代理无法跟随，直到糖再生

### 涌现特征

- 形成**离散的迁徙波**，每波类似一个**连贯对象**（类似 Rule 110 和 Game of Life 的"飞船"）
- ⚠️ **代理只能向北或向东移动，但波却沿对角线（东北）移动！**
- 这是**涌现的典型例子**：群体（波）具有个体代理不具备的性质和行为

## 9.8 涌现 (Emergence)

### 什么是涌现？

**涌现属性**：来自组件间**相互作用**、而非组件自身属性的系统特征。

### 什么是伪涌现？

| 现象 | 是否涌现？ | 原因 |
|------|-----------|------|
| 砖墙坚硬 | ❌ 不是 | 砖和砂浆本身就很硬 → 属于简单的属性叠加 |
| 柔性构件组成刚性结构 | ⚠️ 弱涌现 | 结构属性可从力学定律推导 |
| **Schelling 隔离** | ✅ **强涌现** | 非种族主义的个体产生严重隔离的社会 |
| **Sugarscape 波** | ✅ **强涌现** | 波沿对角线移动，但个体只能南北/东西移动 |
| Sugarscape 财富分布 | ⚠️ 弱涌现 | 可以从视力/代谢/寿命分布合理预测 |

### 涌现与计算不可约性

- **Wolfram 的论点**：传统科学假设"知其规则便可预测其行为"，但许多 CA 是**计算不可约的 (computationally irreducible)**——**没有捷径**，唯一方式是实现/模拟系统本身
- **分析解**：常量时间预测（不依赖时间尺度 t）
- **数值/模拟方法**：时间正比于 t，且对许多系统存在**可预测性的时间上限**
- **推论**：涌现属性**根本上不可预测**，不应期望为复杂系统找到以计算捷径形式存在的"自然法则"

### 哲学争论

- 有人认为"涌现"只是**无知**的代名词——将来理解了就不是涌现
- 本书立场：不应假设涌现属性**永远**无法被还原解释，但也不应假设**一定**能被还原解释
- **计算等价性原理**给出了强有力的理由相信：至少某些涌现属性**绝不可能**被经典还原论模型所"解释"

## 核心概念总结

| 概念 | 核心思想 |
|------|----------|
| **基于代理的模型** | 代理收集信息 → 做决策 → 行动；异质、局部交互、有限信息 |
| **Schelling 模型** | 轻度排外偏好 → 系统级高度隔离（75%+），个体意图 ≠ 系统结果 |
| **Sugarscape** | 简单经济：糖资源 + 视力 + 代谢 + 有限寿命 → 重尾财富分布 |
| **财富不平等** | 即便在极简经济中，不平等也是**固有特征**，难以通过税收等规则避免 |
| **波行为** | 代理只能南北/东西移动，但群体波沿对角线传播——涌现的力证 |
| **涌现** | 来自组件**相互作用**的系统属性，非组件自身属性 |
| **计算不可约性** | 没有预测捷径，唯一方式是运行系统本身 |
| **不可预测性** | 涌现属性可能根本上不可被还原论模型"解释" |
